Миграции

Документация

Миграции - это что-то вроде системы контроля версий для вашей базы данных. Они позволяют команде программистов изменять структуру БД, в то же время оставаясь в курсе изменений других участников. Миграции обычно идут рука об руку с конструктором таблиц для более простого обращения с архитектурой приложения.

Миграции

		php artisan migrate // Запустить миграции
		php artisan migrate:rollback // откат последней миграции
		php artisan migrate:reset // откат всех изменений
		php artisan migrate:refresh // reset + migrate

Генерация миграций

		php make:migration --create=tableName // с созданием таблицы
		php make:migration --table=tableName // для редактирвоания таблицы
		public function up()
{
	Schema::create('tableName', function (Blueprint $table) {
		$table->bigIncrements('id');
		$table->timestamps();
	});
}
Возможные типы полей

Seed

Seeding - механизм наполнения БД начальными данными с помощью специальных классов. Все такие классы хранятся в database/seeds. Они могут иметь любое имя, но следует придерживаться какой-то логики в их именовании - например, UserTableSeeder и т.д. По умолчанию уже определён класс DatabaseSeeder. Из этого класса можно вызывать метод call для подключения других классов с данными, что позволит контролировать порядок их выполнения.

Seed

		php artisan make:seeder UsersTableSeeder
		public function run() {
	User::create([
		'name' => 'admin',
		'email' => 'admin@localhost.com',
		'password' => bcrypt('123456')
	]);
}
		
		DatabaseSeeder.php : $this->call(UsersTableSeeder::class);

Seed

Также необходимо удостовериться, что в файле DatabaseSeeder.php добавлена обработка нужных сидов:

		$this->call(UsersTableSeeder::class);

Аутентицикация

Laravel позволяет сделать аутентификацию очень простой. Фактически, почти всё уже готово к использованию «из коробки». Настройки аутентификации находятся в файле config/auth.php.

Аутентицикация

		php artisan make:auth

"Из коробки" создаются необходимые файлы

  • app/Http/Controllers/Auth
  • app/Http/Middleware/Authentificate.php
  • app/User.php
  • database/migrations/
  • resources/views/auth/
